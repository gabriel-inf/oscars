Axis2 Usage Notes

How to install or upgrade Axis2 + rampart.mar

To Build

The OSCARS code uses the the WSSE functionality provided by the ramapart.mar to digitally sign and verify SOAP messages. So when you install axis2 you must include the rampart.mar.

Download the axis2 distribution from http://ws.apache.org/axis2 and the rampart module from http://ws.apache.org/axis2/modules/rampart/1_3/security-module.html. 
Extract the zip files and follow the instructions in the README.txt  in the rampart-1.3.directory to copy the required libraries to the main "lib" dir of Axis2.

Then in your axis2 build area edit conf/axis2.xml engage rampart and if you want a better chance at debugging errors change DrillDownToRootCause to true and sendStackTraceDetails to true. The last time I built, I also need to comment out the excludes of axis2-adb-codegen*.jar, axis2-codegen*.jar and jibx-bind*.jar from the depends target in the axis2 webapp directory, as they are needed when rampart is invoked. Then build an axis2.war file that will include rampart.mar

You should then copy all the axis2 jar files into OSCARS/lib/axis2 in your OSCARS build area.

To Install
Shutdown tomcat;
Save webapps/axis2/WEB-INF/classes/log4j.properties;
Delete webapps/axis2/WEB-INF/axis2.war and axis2 directory

Copy the new axis.2war file into /usr/local/jarkarta/webapps/
Start Tomcat, which will create the new axis2 directory,
Copy the OSCARS.aar into into /usr/local/jarkarta/webapps/axis2/WEB-INF/services
Copy the saved log4j.properties back into webapps/axis2/WEB-INF/classes/log4j.properties
Restart Tomcat.
