Introduction

The OSCARS Ineterdomain Controller (IDC) allows end users to reserve high priority bandwidth between two hosts. The hosts may both be part of the same ISP domain, or may be on different domains if each of ISPs between the two hosts are running co-operating IDCs.

There are two interfaces provided by the IDC: a forms based web server interface (WBUI) and a web services SOAP message interface (API). Both are secured interfaces and require the user to be registered and authorized by the ISP at which the reservation starts. Individual authorization at subsequent ISPs is not required. Both interfaces allow the user to make, query, list, modify and delete reservations. In addition, the WBUI allows an administrator to add, authorize, and manage users, and the user to modify his user profile, and the API provides interfaces to signal the instantiation of a path or to tear it down. The API also provides interfaces intended for the use of OSCARS services running in other domains to forward a request and to request network topology.

The ESnet WBUI is accessible at https://oscars.es.net/OSCARS/. The I2 WBUI is accessible at https://ndb3-blmt.abilene.ucaid.edu:8443/OSCARS. If you are connecting to I2 through an RON, you will need to contact them for the WBUI URL to use.
