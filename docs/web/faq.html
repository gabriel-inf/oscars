Installation

  • Axis2 and Rampart
    To make Axis2 use Rampart, place conf/rampart-1.3.mar in CATALINA_HOME/webapps/axis2/WEB-INF/modules/ Odds are that this will not work first time. This is because the axis2 1.3 default .war does NOT include some libraries Rampart needs. And, due to Axis2's special classloaders, placing those libraries in the Tomcat shared library folder won't work. Locate the following files: 
    From Axis2 1.3, these .jars under lib: 
	axis2-codegen-1.3.jar, 
	axis-adb-codegen.jar, 
	jibx-bind-1.1.5.jar
	
From rampart-1.3, all the .jars under lib:
	bcprov-jdk15-132.jar    
	opensaml-1.1.jar     
	rampart-core-1.3.jar    
	rampart-trust-1.3.jar   
	rampart-policy-1.3.jar  
	wss4j-1.5.3.jar
	xmlsec-1.4.0.jar
    Copy all the above to CATALINA_HOME/webapps/axis2/WEB-INF/lib/ , and restart Tomcat. Rampart should work now.

Maintenance

  • I ran ant deployall to compile and deploy a new version of the code, but my changes aren't showing! What's wrong?
    Remember to shut down Tomcat and kill -2 the scheduler process before deploying, then start them again.
  • I want to back up OSCARS. How do I do this?
    First, stop Tomcat and the scheduler process. Then, back up the bss and aaa MySQL databases with your favourite MySQL backup tool like mysqldump.
