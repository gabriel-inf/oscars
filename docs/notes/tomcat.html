Tomcat
  apache-tomcat-5.5.17/conf
  
  	service.xml  - defines standalone Tomcat server - catalina
  	     http and/or https connectors are defined here
  	       can define a dumper valve
  	       define remoteAddrValve here to limit the sites that can access the connector
  	  web.xml- defines defaults servlets for static resources, jsp, server-side-includes
  	  context.xml
  	  logging.properties 
  	  tomcat-users.xml  - usernames and passwords
  
  Every Tomcat webapp has a WEB-INF/web.xml file
  
  /usr/local/apache-tomcat-5.5.17/webapps/axis2/WEB-INF/web.xml
  		defines servlets, e.g SOAPMonitor. AdminServlet,Axis2Servlet
  		maps urls to servlet names, and servlet names to class
  		
/usr/local/apache-tomcat-5.5.17/webapps/axis2/WEB-INF/conf/axis2.xml
    Defines parameters like enableMTOM, username, password (for axis admin)
    defnines messageRecievers, transportReceivers(http, tcp, smtp
    Add global modules like SOAPMonitor and rampart
    define phase order (added soapMonitor phase)
     
     
     Client side can use an axis2.xml configuration file, which is needed for signing messages
        One way it is used is by
        ConfigurationContext configContext = ConfigurationContextFactory.createConfigurationContextFromFileSystem(repo, null);
            ServiceClient serviceClient = new ServiceClient(configContext, null)
            
          where repo is a directory that contains a conf/axis2.xml file.
          
           set in axis2.xml  
          org.apache.ws.security.crypto.provider=org.apache.ws.security.components.crypto.Merlin
          org.apache.ws.security.crypto.merlin.keystore.type=jks
          org.apache.ws.security.crypto.merlin.keystore.password=password
          org.apache.ws.security.crypto.merlin.file=sec.jks  (name of keystore file)
          
  	  The security inFlow and outFlow parameters are defined in OSCARS.jar/META-INF/services.xml for the service
