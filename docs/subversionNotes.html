Merging from a branch into the trunk

  1. Make sure you have a copy of the trunk is clean:
                % svn status
  2. Check what the current head version is:
                % svn update
  3. Do a 'dry-run' to see what files will change:
                % svn merge --dry-run -r trunkrevnumber>:headrevnumber ../branches/java
  4. Save this list:
                svn merge --dry-run -r trunkrevnumber>:headrevnumber ../branches/java >~/svn.list
  5. Merge in from the branch (in this case branches/java):
                 svn merge -r  trunkrevnumber>:headrevnumber ../branches/java
  6. Check for any conflicts, these will be marked by a 'C' before the filename. In this example, validator.js has a conflict that we will have to manually resolve:
                 U    web/index.html
             C    web/js/validator.js
  7. After resolving this conflict do:
                svn resolved web/js/validator.js
  8. Then, test with a deploy of the aar and war files on the server.
  9. Commit the changes in the working copy of the trunk:
                svn commit
